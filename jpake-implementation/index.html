    The Yakk JPake Implementation 🔗

    PAKE 🔗

    It's a well-known fact -- strong passwords are better! (up until you can't remember them that is).

    When creating YAKK, I wanted a way of allowing 2 users to share a signalling connection, but without mandating that they use a large secure key to encrypt the connection information. In addition, I was adamant that the controller of the communication channel between these 2 users cannot evesdrop on the messages sent without knowing the shared password, and is not able to brute-force guess the password. (TLDR: see it in action here)

    Enter Password Authenticated Key Exchange (PAKE).

    PAKE allows our 2 parties, Alice and Bob, to establish a shared cryptographic key using a weak shared secret (like a simple password), without needing to trust the communication channel used to establish the shared key.

    JPake using modular arithmetic 🔗

    J-PAKE (Password Authenticated Key Exchange by Juggling) is one such PAKE protocol, proposed by Hao and Ryan.

    It consists of 2 stages:

    • one-time key establishment
    • key confirmation

    A quick summary of the protocol can be written as follows.

    The discrete log problem 🔗

    Let G be a group with binary operation ⊕\oplus⊕.
    Let g be any element of G. For any positive integer k, the expression bkb^kbk denotes performing the operation ⊕\oplus⊕ on bbb with itself kkk times.

    i.e. gk=g⊕g⊕g...⊕g⏟k timesg^k = \underbrace{g \oplus g \oplus g ... \oplus g}_{\text{k times}}gk=k timesg⊕g⊕g...⊕g​​.

    Any integer kkk which solves the equation gk=ag^k = agk=a is termed to be a discrete logarithm of aaa to the base ggg, i.e. k=log⁡gak = \log_gak=logg​a.

    In the above setting, we used the group (Zp)x(Z_p)^x(Zp​)x to perform our cryptographic operations. This is the group of multiplication modulo the prime ppp.

    Eg: In the group 343^434 in the group (Z17)x(Z_{17})^x(Z17​)x, is calculated as: 34mod173^4 mod 1734mod17.

    Note that the discrete log problem is only hard in certain carefully chosen groups. Other than the group of (Zp)x(Z_p)^x(Zp​)x we saw earlier, another group which is useful to us in the cryptographic context is based on elliptic curves.

    Elliptic curves 🔗

    For cryptographic purposes, an elliptic curve is defined as a plane curve over a finite field which consists of points satisfying the equation

    y2=x3+ax+by^2 = x^3 + ax + by2=x3+ax+b

    Combined with a group operation ⊕\oplus⊕ as an addition of 2 elements in the group, allows us to reformulate our discrete log problem as:

    gk=g⊕g⊕g...⊕g⏟k times=g+g+g+...+g⏟k times=kgg^k = \underbrace{g \oplus g \oplus g ... \oplus g}_{\text{k times}} = \underbrace{g +g + g + ... + g}_{\text{k times}} = kggk=k timesg⊕g⊕g...⊕g​​=k timesg+g+g+...+g​​=kg

    We can convert operations from the group (Zp)x(Z_p)^x(Zp​)x to the elliptic curve setting by converting them as the following:

    (Zp)x(Z_p)^x(Zp​)x methodsElliptic curve methods (over a finite field p)
    exponentiation gx mod pg^x \text{ mod } pgx mod p->multiplication xGxGxG
    multiplication gxgy mod pg^xg^y \text{ mod } pgxgy mod p->point addition xG+yGxG + yGxG+yG
    division gxgy mod p\frac{g^x}{g^y} \text{ mod } pgygx​ mod p->point subtraction $(xG - yG)
